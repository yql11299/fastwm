# 部署指南

本文档介绍项目的手动部署方式，支持局域网访问和多服务共存。

## 部署架构

```
                        ┌─────────────────┐
                        │   局域网用户     │
                        └──────┬──────────┘
                               │ http://服务器IP:5173
                        ┌──────▼──────────┐
                        │   PM2 + Node    │ :5173 (前端静态)
                        │   PM2 + Node    │ :3000 (后端API)
                        └──────┬──────────┘
                               │
                        ┌──────▼──────┐
                        │  文件系统    │
                        │  (数据存储)  │
                        └─────────────┘
```

## 环境要求

| 环境 | 要求 |
|------|------|
| Node.js | 18+ |
| npm | 9+ |
| PM2 | 最新版（可通过脚本自动安装）|

## 快速部署

### 方式一：使用部署脚本（推荐）

```bash
# 给脚本执行权限
chmod +x deploy.sh

# 运行部署脚本
./deploy.sh
```

部署脚本会询问：
- 后端 API 端口（默认 `3000`）
- 前端 Web 端口（默认 `5173`）

### 方式二：手动部署

```bash
# 1. 安装 PM2（全局）
npm install -g pm2

# 2. 安装后端依赖
cd server && npm install --production

# 3. 安装前端依赖并构建
cd ../client && npm install && npm run build

# 4. 配置环境变量
cp ../server/.env.example ../server/.env

# 5. 使用 PM2 启动
cd ..
pm2 start ecosystem.config.js

# 6. 保存并设置开机自启
pm2 save
pm2 startup
```

## 访问服务

部署完成后访问：

```
http://服务器IP:5173
```

例：`http://192.168.1.100:5173`

## 端口配置

### 修改端口

**方式一：重新运行部署脚本**

```bash
./deploy.sh
```

**方式二：手动修改**

```bash
# 修改后端端口
sed -i 's/^PORT=.*/PORT=3001/' server/.env

# 重启服务
pm2 restart all
```

### 常见端口冲突

| 端口 | 常见冲突服务 | 建议替代端口 |
|------|-------------|-------------|
| 3000 | Node.js 应用 | 3001, 8080 |
| 5173 | Vite 开发服务器 | 5174, 8080 |

## 局域网访问配置

### 1. 确保防火墙开放

```bash
# Ubuntu/Debian
sudo ufw allow 5173/tcp

# CentOS/RHEL
sudo firewall-cmd --permanent --add-port=5173/tcp
sudo firewall-cmd --reload
```

### 2. 获取服务器 IP

```bash
hostname -I
```

### 3. 访问服务

在局域网内其他机器上访问：`http://192.168.1.100:5173`

## 多服务共存

### 场景一：已有 80/443 端口服务

使用非标准端口（如 5173）访问本服务，不影响现有服务。

### 场景二：使用 Nginx 反向代理

如需通过标准端口访问，可配置 Nginx 反向代理：

```nginx
server {
    listen 80;
    server_name watermark.example.com;

    location / {
        proxy_pass http://localhost:5173;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /api/ {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 目录结构

部署后目录：

```
fastwm/
├── ecosystem.config.js    # PM2 配置
├── deploy.sh             # 部署脚本
├── server/               # 后端服务
│   ├── src/             # 后端源码
│   └── .env             # 环境变量
├── client/               # 前端
│   ├── dist/            # 构建产物（部署后生成）
│   └── src/             # 前端源码

    ├── users/           # 用户数据
    ├── documents/       # 证件文档
    ├── exports/         # 导出的水印文件
    └── backgrounds/     # 背景图片
```

## 常用命令

```bash
# 查看服务状态
pm2 list

# 查看日志
pm2 logs

# 查看指定服务日志
pm2 logs fastwm-server
pm2 logs fastwm-client

# 重启服务
pm2 restart all

# 重启指定服务
pm2 restart fastwm-server

# 停止服务
pm2 stop all

# 删除服务
pm2 delete all

# 保存进程列表（重启后恢复）
pm2 save

# 清除保存的进程列表
pm2 cleardump
```

## 故障排查

### 服务无法启动

```bash
# 查看详细日志
pm2 logs --lines 100

# 检查端口占用
netstat -tlnp | grep 3000
netstat -tlnp | grep 5173
```

### 前端无法访问 API

```bash
# 确认后端健康
curl http://localhost:3000/health

# 检查后端日志
pm2 logs fastwm-server
```

### PM2 进程消失

```bash
# 查看所有进程（包括已停止）
pm2 list --all

# 恢复进程
pm2 resurrect

# 或重新启动
pm2 start ecosystem.config.js
```

## 升级

cd client && npm run build && cd ..

pm2 restart all
