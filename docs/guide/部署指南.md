# 部署指南

本文档介绍项目的 Docker 部署方式，支持局域网访问和多服务共存。

## 部署架构

```
                        ┌─────────────────┐
                        │   局域网用户     │
                        └──────┬──────────┘
                               │ http://服务器IP:8080
                        ┌──────▼──────────┐
                        │   Nginx        │ :8080 (默认)
                        │  (反向代理)     │
                        └──────┬──────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
       ┌──────▼──────┐  ┌──────▼──────┐
       │   前端       │  │   后端      │
       │  (静态资源)  │  │  :3000      │
       └─────────────┘  └──────┬──────┘
                               │
                        ┌──────▼──────┐
                        │  文件系统    │
                        │  (数据存储)  │
                        └─────────────┘
```

## 环境要求

| 环境 | 要求 |
|------|------|
| Docker | 20.10+ |
| Docker Compose | 2.0+ |

## 快速部署

### 方式一：使用部署脚本（推荐）

```bash
# 给脚本执行权限
chmod +x deploy.sh

# 运行部署脚本
./deploy.sh
```

部署脚本会询问：
- 后端 API 端口（默认 `3000`）
- 前端 Web 端口（默认 `8080`）

### 方式二：手动部署

```bash
# 构建并启动（使用默认端口）
docker-compose up -d

# 或指定端口
SERVER_PORT=3001 CLIENT_PORT=9090 docker-compose up -d
```

## 访问服务

部署完成后访问：

```
http://服务器IP:8080
```

例：`http://192.168.1.100:8080`

## 端口配置

### 修改端口

**方式一：环境变量**

```bash
export SERVER_PORT=3000
export CLIENT_PORT=8080
docker-compose up -d
```

**方式二：直接编辑 docker-compose.yml**

```yaml
services:
  server:
    ports:
      - "3001:3000"  # 修改左侧宿主端口
  client:
    ports:
      - "8081:80"    # 修改左侧宿主端口
```

### 常见端口冲突

| 端口 | 常见冲突服务 | 建议替代端口 |
|------|-------------|-------------|
| 80/443 | Apache, Nginx, IIS | 8080, 8443 |
| 3000 | Node.js 应用 | 3001, 8080 |

## 局域网访问配置

### 1. 确保防火墙开放

```bash
# Ubuntu/Debian
sudo ufw allow 8080/tcp

# CentOS/RHEL
sudo firewall-cmd --permanent --add-port=8080/tcp
sudo firewall-cmd --reload
```

### 2. 获取服务器 IP

```bash
hostname -I
```

### 3. 访问服务

在局域网内其他机器上访问：`http://192.168.1.100:8080`

## 多服务共存

### 场景一：已有 Nginx 服务

使用不同端口或配置反向代理。参见上方端口配置。

### 场景二：使用子路径

如需通过子路径访问（如 `http://server/watermark/`），需修改 Nginx 配置并重新构建镜像。

## 目录结构

部署后目录：

```
fastwm/
├── docker-compose.yml      # 服务编排配置
├── Dockerfile.server       # 后端构建文件
├── Dockerfile.client       # 前端构建文件
├── nginx.conf             # Nginx 配置
├── deploy.sh              # 部署脚本
│
├── data/                  # 持久化数据
│   ├── users/            # 用户数据
│   ├── documents/        # 证件文档
│   ├── exports/          # 导出的水印文件
│   └── backgrounds/      # 背景图片
│
├── fonts/                 # 字体文件（TTF/OTF）
│   ├── simhei.ttf        # 黑体
│   └── simkai.ttf        # 楷体
│
└── server/               # 后端源码
```

## 常用命令

```bash
# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 重启服务
docker-compose restart

# 停止服务
docker-compose down

# 重新构建
docker-compose build --no-cache
docker-compose up -d

# 进入容器调试
docker exec -it fastwm-server sh
```

## 故障排查

### 服务无法启动

```bash
# 检查端口占用
netstat -tlnp | grep 3000
netstat -tlnp | grep 8080

# 查看日志
docker-compose logs
```

### 前端无法访问 API

```bash
# 确认后端健康
curl http://localhost:3000/health
```

## 升级

```bash
git pull
docker-compose build --no-cache
docker-compose up -d
```

## 卸载

```bash
docker-compose down
# 如需删除数据：rm -rf data/
```
