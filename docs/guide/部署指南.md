# 部署指南

本文档介绍项目的部署方式，支持局域网访问和多服务共存。

## 环境要求

| 环境 | 要求 |
|------|------|
| Node.js | 18+ |
| npm | 9+ |
| PM2 | 最新版 |

---

## 第一步：安装 Node.js

### Windows 系统

1. 打开浏览器访问 https://nodejs.org/

2. 点击绿色的 **LTS 版本** 下载按钮（约 30MB）

3. 下载完成后双击安装文件

4. 安装向导中：
   - 点击 **Next**
   - 接受许可协议，点击 **Next**
   - 安装路径保持默认，点击 **Next**
   - **重要**：勾选 "**Automatically install necessary tools**"
   - 点击 **Install**
   - 点击 **Finish**

5. 验证安装：
   - 按 `Win + R`，输入 `cmd`，回车
   - 在黑色窗口中输入 `node -v`，回车
   - 应该显示 `v20.x.x` 或类似版本
   - 再输入 `npm -v`，回车
   - 应该显示版本号

### Linux 系统（Ubuntu/Debian）

打开终端，执行：

```bash
# 更新软件源
sudo apt update

# 安装 Node.js 20.x 版本
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证安装
node -v
npm -v
```

### Linux 系统（CentOS/RHEL）

打开终端，执行：

```bash
# 安装 Node.js 20.x 版本
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs

# 验证安装
node -v
npm -v
```

### macOS 系统

打开终端，执行：

```bash
# 如果没有 brew，先安装 Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装 Node.js
brew install node@20

# 验证安装
node -v
npm -v
```

## 第二步：安装 PM2

PM2 是一个进程管理器，用于保持服务运行。

### Windows 系统

1. 按 `Win + R`，输入 `cmd`，回车

2. 在命令行中输入：

```bash
npm install -g pm2
```

3. 验证安装：
```bash
pm2 -v
```

### Linux / macOS 系统

在终端中输入：

```bash
npm install -g pm2

# 验证安装
pm2 -v
```

## 第三步：部署应用

### 方式一：使用部署脚本（推荐）

1. 将项目文件复制到服务器（假设在 `/home/fastwm` 或 `C:\fastwm`）

2. 打开终端，进入项目目录：
```bash
# Windows
cd C:\fastwm

# Linux/macOS
cd /home/fastwm
```

3. 给脚本执行权限：
```bash
# Linux/macOS 终端
chmod +x deploy.sh
```

4. 运行部署脚本：
```bash
./deploy.sh
```

5. 脚本会询问端口配置，直接回车使用默认端口（3000 和 5173）

### 方式二：手动部署

1. 将项目文件复制到服务器

2. 打开终端，进入项目目录

3. 安装后端依赖：
```bash
cd server
npm install --production
cd ..
```

4. 安装前端依赖并构建：
```bash
cd client
npm install
npm run build
cd ..
```

5. 配置环境变量：
```bash
cp server/.env.example server/.env
```

6. 使用 PM2 启动：
```bash
pm2 start ecosystem.config.js
```

7. 保存进程列表并设置开机自启：
```bash
pm2 save
pm2 startup
```

## 访问服务

部署完成后，在浏览器中访问：

```
http://服务器IP:5173
```

获取服务器 IP 的方法：
- **Windows**：在命令行中输入 `ipconfig`，查找 "IPv4 地址"
- **Linux**：在终端中输入 `hostname -I`
- **macOS**：在终端中输入 `hostname -I`

例：`http://192.168.1.100:5173`

## 端口配置

默认端口：
- 后端 API：`3000`
- 前端 Web：`5173`

### 修改端口

**方式一：重新运行部署脚本**

```bash
./deploy.sh
```

**方式二：手动修改**

1. 编辑 `server/.env` 文件：
```bash
# Windows 使用 notepad
notepad server/.env

# Linux/macOS 使用 nano
nano server/.env
```

2. 修改 `PORT=3000` 为新的端口（如 `PORT=3001`）

3. 重启服务：
```bash
pm2 restart all
```

### 常见端口冲突

| 端口 | 常见冲突服务 | 建议替代端口 |
|------|-------------|-------------|
| 3000 | Node.js 应用 | 3001, 8080 |
| 5173 | Vite 开发服务器 | 5174, 8080 |

## 防火墙配置（允许局域网访问）

### Windows 系统

1. 按 `Win + R`，输入 `wf.msc`，回车

2. 点击左侧 "**入站规则**"

3. 点击右侧 "**新建规则**"

4. 选择 "**端口**"，点击 "**下一步**"

5. 输入 `5173`，点击 "**下一步**"

6. 选择 "**允许连接**"，点击 "**下一步**"

7. 保持默认选项，点击 "**下一步**"

8. 名称填写 "fastwm"，点击 "**完成**"

### Linux 系统（Ubuntu/Debian）

```bash
sudo ufw allow 5173/tcp
sudo ufw allow 3000/tcp
sudo ufw reload
```

### Linux 系统（CentOS/RHEL）

```bash
sudo firewall-cmd --permanent --add-port=5173/tcp
sudo firewall-cmd --permanent --add-port=3000/tcp
sudo firewall-cmd --reload
```

## 常用命令

```bash
# 查看服务状态
pm2 list

# 查看日志
pm2 logs

# 查看指定服务日志
pm2 logs fastwm-server
pm2 logs fastwm-client

# 重启服务
pm2 restart all

# 停止服务
pm2 stop all

# 删除所有服务
pm2 delete all

# 保存进程列表
pm2 save

# 查看开机自启任务（不需要手动执行）
pm2 startup
```

## 故障排查

### 服务无法启动

```bash
# 查看详细日志
pm2 logs --lines 100

# 检查端口占用
netstat -tlnp | grep 3000
netstat -tlnp | grep 5173
```

### 前端无法访问 API

```bash
# 确认后端健康
curl http://localhost:3000/health

# 检查后端日志
pm2 logs fastwm-server
```

### PM2 进程消失（重启后）

```bash
# 恢复进程
pm2 resurrect

# 或重新启动
pm2 start ecosystem.config.js
```

## 目录结构

部署后目录：

```
fastwm/
├── ecosystem.config.js    # PM2 配置
├── deploy.sh             # 部署脚本
├── server/               # 后端服务
│   ├── src/             # 后端源码
│   └── .env             # 环境变量
├── client/               # 前端
│   ├── dist/            # 构建产物（部署后生成）
│   └── src/             # 前端源码
└── data/                 # 持久化数据
    ├── users/           # 用户数据
    ├── documents/       # 证件文档
    ├── exports/         # 导出的水印文件
    └── backgrounds/     # 背景图片
```

## 升级

```bash
# 拉取新代码（如果有版本控制）
git pull

# 重新构建前端
cd client && npm run build && cd ..

# 重启服务
pm2 restart all
```

## 卸载

```bash
# 停止并删除所有进程
pm2 stop all
pm2 delete all

# 删除代码目录（可选）
# Windows
# rmdir /s /q fastwm

# Linux/macOS
# rm -rf fastwm/

# 注意：data/ 目录如需保留，请勿删除
```
