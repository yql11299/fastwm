# 部署指南

本文档介绍项目的三种部署方式，支持局域网访问和多服务共存。

---

## 部署方式对比

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **Docker 部署** | 一键部署，隔离环境，迁移简单 | 需要 Docker 环境，国内镜像可能较慢 | 熟悉 Docker 的用户 |
| **PM2 部署** | 资源占用少，无需 Docker，开机自启 | 需要手动安装 Node.js 和 PM2 | 推荐用于国内服务器 |
| **Node.js 原生** | 最简单，占用资源最少 | 无进程管理，重启后需手动启动 | 开发测试环境 |

**推荐**：国内服务器使用 **PM2 部署**，避免 Docker 镜像拉取问题。

---

## 方式一：Docker 部署

### 环境要求

| 环境 | 要求 |
|------|------|
| Docker | 20.10+ |
| Docker Compose | 2.0+ |

### 安装 Docker

#### Windows 系统

1. 下载 Docker Desktop：https://www.docker.com/products/docker-desktop/

2. 安装并启动 Docker Desktop

3. 等待托盘图标显示 "Docker Desktop is running"

#### Linux 系统（Ubuntu/Debian）

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 添加当前用户到 docker 组（避免每次用 sudo）
sudo usermod -aG docker $USER

# 启动 Docker 服务
sudo systemctl start docker
sudo systemctl enable docker
```

#### Linux 系统（CentOS/RHEL）

```bash
sudo yum install -y docker
sudo systemctl start docker
sudo systemctl enable docker
```

#### macOS 系统

```bash
brew install --cask docker
open -a Docker
```

### 使用一键部署脚本

```bash
# Windows：双击运行 deploy-docker.bat
# 或在命令行中运行
deploy-docker.bat

# Linux/macOS：
chmod +x deploy-docker.sh
./deploy-docker.sh
```

脚本会：
1. 检查 Docker 环境
2. 清理旧容器和镜像
3. 拉取/构建镜像
4. 启动服务

### 手动 Docker 部署

```bash
# 清理旧容器（可选）
docker-compose down

# 构建并启动
docker-compose up -d

# 查看状态
docker-compose ps
```

### 访问服务

```
http://服务器IP:8080
```

### 常用命令

```bash
# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 重启服务
docker-compose restart

# 停止服务
docker-compose down

# 重新构建
docker-compose build --no-cache
docker-compose up -d
```

## 方式二：PM2 部署（推荐）

PM2 是进程管理器，支持开机自启、日志管理、进程守护。适用于真实 Linux 服务器部署。

### 环境要求

| 环境 | 要求 |
|------|------|
| Node.js | 18+ |
| npm | 9+ |
| PM2 | 最新版 |

### 安装 Node.js

#### Linux 系统（Ubuntu/Debian）

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

#### Linux 系统（CentOS/RHEL）

```bash
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs
```

#### macOS 系统

```bash
brew install node@20
```

#### Windows 系统

1. 访问 https://nodejs.org/
2. 下载 LTS 版本
3. 双击安装，保持默认选项

### 使用一键部署脚本

```bash
# Linux/macOS：
chmod +x deploy-pm2.sh
./deploy-pm2.sh

# Windows：
deploy-pm2.bat
```

脚本会：
1. 检查 Node.js 和 npm
2. 安装 PM2（如未安装）
3. 自动安装 canvas 编译依赖（Linux）
4. 配置 npm 镜像（国内加速）
5. 清理旧的 node_modules 和 dist
6. 安装前后端依赖
7. 构建前端（自动使用本机 IP 作为 API 地址）
8. 启动服务并配置开机自启

### 脚本特性

| 特性 | 说明 |
|------|------|
| 局域网访问 | 自动检测本机 IP，服务可被局域网内其他机器访问 |
| canvas 依赖 | 自动安装 libcairo2-dev 等编译依赖 |
| npm 镜像 | 自动配置 npmmirror.com 加速 |
| 端口配置 | 支持自定义后端/前端端口 |
| 进程管理 | 自动配置 PM2 开机自启 |

### 手动 PM2 部署

```bash
# 安装 PM2
npm install -g pm2

# 安装 canvas 编译依赖（Debian/Ubuntu）
sudo apt-get install -y build-essential python3 libcairo2-dev libpango1.0-dev libjpeg-dev libgif-dev librsvg2-dev

# 安装后端依赖
cd server
npm install

# 配置环境变量
cp .env.example .env

# 安装前端依赖并构建
cd ../client
npm install
API_URL="http://本机IP:3000/api" npm run build

# 启动服务
cd ..
pm2 start ecosystem.config.js

# 保存进程列表
pm2 save

# 设置开机自启
pm2 startup
```

### 常用命令

```bash
# 查看服务状态
pm2 list

# 查看日志
pm2 logs

# 重启服务
pm2 restart all

# 停止服务
pm2 stop all

# 删除所有服务
pm2 delete all

# 保存进程列表
pm2 save
```

### 访问服务

```
http://服务器IP:5173
```

局域网内其他机器访问：`http://服务器局域网IP:5173`

> **注意**：如果服务端是 WSL 环境，Windows 浏览器需通过端口转发访问（`127.0.0.1:5173`），具体请参考 WSL 文档。

## 方式三：Node.js 原生部署

适合开发测试环境，不推荐用于生产环境。

### 环境要求

| 环境 | 要求 |
|------|------|
| Node.js | 18+ |

### 安装 Node.js

参见上方 PM2 部署的 Node.js 安装步骤。

### 使用一键部署脚本

```bash
# Windows：双击运行 deploy-node.bat
# 或在命令行中运行
deploy-node.bat

# Linux/macOS：
chmod +x deploy-node.sh
./deploy-node.sh
```

脚本会：
1. 检查 Node.js
2. 清理旧的构建产物
3. 安装依赖
4. 构建前端
5. 启动服务

### 手动原生部署

```bash
# 终端 1：启动后端
cd server
npm install
npm start

# 终端 2：启动前端
cd client
npm install
npm run dev
```

### 访问服务

```
前端：http://localhost:5173
后端：http://localhost:3000
```

## 懒人一键部署脚本

项目提供三个一键部署脚本：

| 脚本 | 适用系统 | 部署方式 |
|------|----------|----------|
| `deploy-docker.sh` / `deploy-docker.bat` | Linux/macOS / Windows | Docker |
| `deploy-pm2.sh` / `deploy-pm2.bat` | Linux/macOS / Windows | PM2 |
| `deploy-node.sh` / `deploy-node.bat` | Linux/macOS / Windows | Node.js 原生 |

### 使用方法

**Windows 用户：**
1. 下载项目到服务器
2. 双击对应的 .bat 文件
3. 按提示操作

**Linux/macOS 用户：**
```bash
chmod +x deploy-*.sh
./deploy-xxx.sh
```

## 端口配置

默认端口：

| 服务 | 默认端口 |
|------|----------|
| 后端 API | 3000 |
| 前端 Web（PM2/原生） | 5173 |
| 前端 Web（Docker） | 8080 |

### 修改端口

**Docker 部署：**
```bash
# 修改 docker-compose.yml 中的端口映射
# 或运行时指定
SERVER_PORT=3001 CLIENT_PORT=9090 docker-compose up -d
```

**PM2 部署：**
1. 编辑 `server/.env` 文件
2. 修改 `PORT=3000` 为新端口
3. 重启服务：`pm2 restart all`

### 常见端口冲突

| 端口 | 常见冲突服务 | 建议替代端口 |
|------|-------------|-------------|
| 3000 | Node.js 应用 | 3001, 8080 |
| 5173 | Vite 开发服务器 | 5174, 8080 |
| 8080 | 其他 Web 服务 | 8081, 9090 |

## 防火墙配置（允许局域网访问）

### Windows 系统

```powershell
# 使用 PowerShell（管理员）
netsh advfirewall firewall add rule name="fastwm" dir=in action=allow protocol=tcp localport=5173,3000
```

### Linux 系统（Ubuntu/Debian）

```bash
sudo ufw allow 5173/tcp
sudo ufw allow 3000/tcp
sudo ufw reload
```

### Linux 系统（CentOS/RHEL）

```bash
sudo firewall-cmd --permanent --add-port=5173/tcp
sudo firewall-cmd --permanent --add-port=3000/tcp
sudo firewall-cmd --reload
```

## 故障排查

### Docker 部署问题

```bash
# 查看日志
docker-compose logs -f

# 检查容器状态
docker-compose ps

# 清理后重新构建
docker-compose down
docker-compose build --no-cache
docker-compose up -d
```

### PM2 部署问题

```bash
# 查看日志
pm2 logs --lines 100

# 检查端口占用
netstat -tlnp | grep 3000
netstat -tlnp | grep 5173

# 重启服务
pm2 restart all
```

### Node.js 原生部署问题

```bash
# 确认后端健康
curl http://localhost:3000/health

# 检查前端构建是否成功
ls client/dist/
```


```bash
git pull
```
