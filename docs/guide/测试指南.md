# 测试指南

本文档介绍项目的测试框架和运行方法。

## 测试架构

```
┌─────────────────────────────────────────────────────────┐
│                    测试金字塔                            │
├─────────────────────────────────────────────────────────┤
│                    ┌───────────┐                        │
│                    │   E2E     │  ← Playwright          │
│                    │  测试     │                        │
│                    └─────┬─────┘                        │
│                          │                              │
│              ┌───────────┴───────────┐                  │
│              │     集成测试          │                  │
│              └─────┬──────────────┘                  │
│                    │                                  │
│         ┌──────────┴──────────┐                       │
│         │      单元测试        │                       │
│         └─────────────────────┘                       │
└─────────────────────────────────────────────────────────┘
```

## 测试框架

| 层级 | 框架 | 命令 |
|------|------|------|
| 前端单元 | Vitest + RTL | `npm test` |
| E2E | Playwright | `npm run test:e2e` |
| 后端 | Jest + Supertest | `npm test` |

## 快速开始

### 前端测试

```bash
cd client
npm test
```

### E2E 测试

```bash
cd client
npm run test:e2e
```

### 后端测试

```bash
cd server
npm test
```

## 编写测试

### 前端组件测试

```jsx
import { render, screen } from '@testing-library/react';
import MyComponent from './MyComponent';

test('renders correctly', () => {
  render(<MyComponent />);
  expect(screen.getByText('Hello')).toBeInTheDocument();
});
```

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| 测试超时 | 增加 `test.timeout()` |
| Mock 不生效 | 检查 mock 路径是否正确 |
