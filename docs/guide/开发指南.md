# 开发指南

本文档介绍本地开发环境搭建、代码组织结构和调试方法。

## 环境要求

| 工具 | 版本 | 说明 |
|------|------|------|
| Node.js | 18+ | 后端和前端运行的基础 |
| npm | 9+ | 包管理工具 |
| Git | 2.30+ | 版本控制 |

## 项目结构

```
fastwm/
├── client/                 # 前端 (React + Vite)
│   ├── src/
│   │   ├── api/           # API 客户端
│   │   ├── components/    # React 组件
│   │   ├── hooks/         # 自定义 Hooks
│   │   ├── stores/        # Zustand 状态管理
│   │   └── utils/         # 工具函数
│   └── tests/             # 测试文件
│
├── server/                 # 后端 (Node.js + Express)
│   ├── src/
│   │   ├── routes/       # API 路由
│   │   ├── services/     # 业务逻辑
│   │   ├── middleware/   # 中间件
│   │   ├── utils/        # 工具函数
│   │   └── config/       # 配置
│   └── data/              # 数据存储
│
└── fonts/                 # 字体文件 (TTF/OTF)
```

## 快速开始

### 1. 安装依赖

```bash
# 安装前端依赖
cd client && npm install

# 安装后端依赖
cd ../server && npm install
```

### 2. 启动开发服务器

```bash
# 终端 1: 启动后端
cd server && npm run dev

# 终端 2: 启动前端
cd client && npm run dev
```

访问 http://localhost:5173

## 代码规范

### Git 提交规范

```
feat: 新功能
fix: 修复 Bug
refactor: 重构
docs: 文档更新
test: 测试相关
chore: 构建/工具
```

### 文件命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件 | PascalCase | `Canvas.jsx` |
| Hooks | camelCase，use 前缀 | `useCanvas.js` |
| 工具函数 | camelCase | `watermarkRenderer.js` |
